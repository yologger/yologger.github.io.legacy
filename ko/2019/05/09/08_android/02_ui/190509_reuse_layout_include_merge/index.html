계층구조가 복잡한 UI(중첩된 ViewGroup)는 성능을 저하시킵니다. 따라서 중첩된 ViewGroup은 최대한 제거하는 것이 좋습니다.

<merge />를 사용하면 RelativeLayout, ConstraintLayout, LinearLayout같은 ViewGroup을 사용하지 않고 루트 뷰를 구성할 수 있습니다.

<merge />는 UI의 중첩도를 낮추지만 RelativeLayout, ConstraintLayout, LinearLayout처럼 원하는 곳에 자식 뷰를 배치할 수 없습니다. 따라서 상황에 맞게 <merge />와 <include />를 사용해야합니다.
