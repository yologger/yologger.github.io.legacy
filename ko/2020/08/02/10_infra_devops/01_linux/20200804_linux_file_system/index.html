[Linux] 리눅스 파일 시스템 구조

2020/08/02 카테고리 Infra, DevOps, Linux

거의 대부분의 리눅스 배포판은 리눅스 파일시스템 계층 표준인 Linux Filesystem Hierarchy Standard을 준수한다. 따라서 다음 디렉토리 구조를 가진다.

여기서 중요한 몇 가지 디렉토리를 살펴보자.

이름 설명
/ 최상위 디렉토리인 루트 디렉토리
/boot/ 부팅에 필요한 파일들이 존재한다.
/bin/ 모든 사용자들이 사용할 수 있는 명령어 바이너리. ls, cat, cp, rm, ps 같은 명령어가 이 곳에 존재한다.
/sbin/ 시스템 관리자가 사용하는 시스템 명령어 바이너리. useradd, userdel, usermod 같은 명령어가 이 곳에 존재한다.
/lib /bin/과 /sbin의 명령어 바이너리가 필요로하는 라이브러리 파일
/root 루트 사용자의 홈 디렉토리
/home 일반 사용자의 홈 디렉토리
/usr Universal Resource의 약자. 일반 사용자가 사용하는 모든 프로그램과 자원 파일들이 존재한다.
/etc 시스템 전반의 환경설정 파일이 존재한다.
/var 상대적으로 자주 변하는 파일들이 존재한다.
/tmp 다양한 프로그램에서 만들어지는 임시 파일들이 저장된다. 시스템이 재부팅될 때 마다 이 디렉토리가 비워진다.
/mnt 테스트2
/media USB 드라이버, CD-ROM 같은 휴대용 장치가 연결될 때의 마운트 포인트가 이 곳에 존재한다.
/dev 커널이 장치하는 모든 장치. 리눅스는 장치도 파일로 관리한다.
/opt Optional의 약자로
/proc 리눅스에서 동작 중인 프로세스, 하드웨어, 시스템 등의 상태 정보를 담은 파일이 저장된다.
