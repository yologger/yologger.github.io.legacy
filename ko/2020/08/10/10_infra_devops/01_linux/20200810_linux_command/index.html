[Linux] 리눅스 명령어 정리

2020/08/10 카테고리 Infra, DevOps, Linux

목차

  • 명령어 종류
    1. 실행 파일
    2. 쉘에 내장된 명령어
    3. 쉘 함수
    4. 별칭
  • 명령어 정리 (알파벳 순)
    • alias - 별칭 만들기
    • cal - 달력과 현재 날짜를 표시
    • cp - 파일이나 디렉토리를 복사
    • date - 현재 시간과 날짜를 표시
    • df - 디스크 관련 정보를 표시
    • diff - 두 파일의 내용을 비교
    • echo - 텍스트를 화면에 출력
    • file - 파일 타입을 표시
    • find - 파일 탐색
    • free - 메모리 관련 정보를 표시
    • head - 파일의 앞 부분을 출력
    • help - 명령어 도움말
    • less - 파일 내용 확인
    • mkdir - 디렉토리 생성
    • more - 내용이 긴 파일 확인
    • mv - 파일 또는 디렉토리를 이동하거나 이름 변경
    • ping - 패킷 전송하기
    • ps - 프로세스 확인
    • rmdir - 디렉토리 삭제
    • tail - 파일의 뒷 부분을 출력
    • tar - 파일 압축
    • top - 프로세스 등 시스템 상태를 동적으로 확인
    • which - 실행 파일의 설치 위치 확인

명령어 종류

리눅스 명령어는 크게 네 종류로 구분된다. type명령어를 사용하면 명령어의 타입을 확인할 수 있다.

(1) 실행 파일

/usr/bin 디렉토리에서 본 파일들처럼 실행 파일을 말한다.

(2) 쉘에 내장된 명령어

쉘에 내장된 명령어다.

(3) 쉘 함수

(4) 별칭

별칭을 사용하면 명령어로 본인의 명령어를 새롭게 정의할 수 있다.

명령어 정리

alias

alias명령어를 사용하면 우리만의 명령어를 만들 수 있다.

우선 type명령어로 alias가 존재하는지 확인한다.

이제 다음과 같은 형태로 alias를 생성할 수 있다.

다음과 같이 모든 alias를 확인할 수 있다.

unalias명령어로 alias를 삭제할 수 있다.

cal

달력과 현재 날짜를 표시한다.

cp

파일이나 디렉토리를 복사할 때 사용한다.

file1을 file2에 복사한다.

file을 directory에 복사한다.

디렉토리의 모든 파일을 다른 디렉토리에 복사한다.

date

현재 시간과 날짜를 표시한다.

df

현재 사용 중인 디스크 정보와 사용 가능한 디스크의 용량을 표시한다.

diff

두 파일을 비교할 수 있다.

echo

텍스트를 화면에 출력한다.

file

파일의 타입을 확인할 수 있다.

find

파일을 탐색할 수 있다. 보통 다음과 같은 형식으로 사용한다.

현재 디렉토리와 하위 디렉토리에서 txt로 끝나는 파일을 탐색한다.

사용자의 홈 디렉토리와 하위 디렉토리에서 txt로 끝나는 파일을 탐색한다.

free

메모리 사용 현황을 확인할 수 있다.

head

파일의 앞 부분을 출력한다.

help

명령어의 도움말 기능이다. 설명 및 사용법을 확인할 수 있다.

less

텍스트 파일의 내용을 확인할 수 있다.

mkdir

디렉토리를 생성할 수 있다.

more

파일 내용이 너무 길면 스크롤로 확인해야하는 불편함이 있다. more는 파일 내용을 한 페이지씩 보여준다.

mv

파일 또는 디렉토리를 이동하거나 이름을 변경할 때 사용한다.

파일 이름을 변경한다.

파일을 디렉토리로 이동한다.

현재 디렉토리의 모든 파일을 다른 디렉토리로 이동한다. 단 타겟 디렉토리가 생성되어있어야 한다.

ping

목적지로 패킷을 전송한다. 목적지와의 네트워크 연결을 확인하는데 사용할 수 있다.

ps

프로세스 정보를 확인할 수 있다.

보통 aux옵션을 함께 사용한다.

tar

파일을 압축할 수 있다. -cvf옵션을 사용하면 된다.

xvf옵션으로 압축을 풀 수 있다.

top

프로세스 등 시스템 상태를 동적으로 확인할 수 있다.

ps가 명령어 실행 순간의 스냅샷을 보여준다면 top는 동적으로 시스템 활동을 보여준다.

rmdir

디렉토리를 삭제한다.

tail

파일의 뒷 부분을 출력한다.

-f옵션을 추가하면 실시간으로 파일을 확인할 수 있다. 로그 파일의 최근 내용을 확인하는데 유용하다.

which

실행 파일의 설치 위치를 확인할 수 있다.
