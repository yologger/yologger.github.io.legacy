GitHub을 통한 협업 과정

GitHub을 통한 일반적인 협업 과정은 다음과 같다.

  1. 보통 프로젝트에 처음 투입되는 개발자는 git clone명령어를 사용하여 로컬 PC에 프로젝트를 구성한다.
  2. A 개발자는 작업을 마치고 git push를 통해 변경 내용을 원격 저장소에 반영시킨다.
  3. B 개발자는 작업을 시작하기 전, 다른 개발자가 원격 저장소에 반영한 변경 내용을 본인의 로컬 PC에 반영해야한다. 이 때 git pull 명령어를 사용한다.
