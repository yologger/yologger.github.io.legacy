[Java] 직렬화와 transient

2019/01/03 카테고리 Java

직렬화

운영체제나 네트워크에는 Java의 객체의 개념이 없다. 그렇다면 어떻게 객체를 운영체제의 파일 시스템에 저장하거나 네트워크로 전송할 수 있을까?

직렬화(Serialization)는 객체를 바이트로 변환하여 저장하는 기술을 의미한다. 역직렬화(Deserialization)는 바이트를 다시 Java 객체로 변환하는 것이다.

Serializable

직렬화할 클래스는 Serializable를 구현해야한다.

ObjectInputStream, ObjectOutputStream

직렬화와 역직렬화에는 보통 ObjectInputStream, ObjectOutputStream을 사용한다.

transient

멤버변수에 transient을 붙이면 직렬화 대상에서 제외된다.

transient가 붙은 멤버변수를 역직렬화하면 값이 null이 된다.

[Java] 입출력

2019/01/02 카테고리 Java

애플리케이션에서는 외부에서 데이터를 읽어오거나 외부로 데이터를 보내는 작업이 빈번하게 일어난다. 외부에서 애플리케이션으로 데이터를 읽어오는 것을 입력(Input), 애플리케이션에서 외부로 데이터를 내보내는 것을 출력(Output)이라고 한다.

입력

  • 사용자로부터 키보드, 마우스 등으로 데이터를 받아옴
  • 파일에서 데이터를 읽어옴
  • 네트워크를 통해 데이터를 읽어옴

출력

  • 모니터에 데이터를 출력하여 보여줌
  • 파일에 데이터를 출력하여 저장함

Java에서는 입출력을 위한 요소들이 java.io 패키지에 포함되어있다.

InputStream, OutputStream

Java에서는 InputStream을 통해 입력을, OutputStream을 통해 출력을 처리한다. InputStream과 OutputStream은 1바이트(8비트) 단위로 데이터를 입출력한다. 문자 뿐만 아니라 이미지, 멀티미디어 등 모든 종류의 데이터를 주고받을 수 있다.

InputStream과 OutputStream은 인터페이스며 대표적인 구현체는 다음과 같다.

  • FileInputStream, FileOutputStream
  • DataInputStream, DataOutputStream
  • ObjectInputStream, ObjectOutputStream

(1) FileInputStream, FileOutputStream

FileInputStream, FileOutputStream은 파일에 입출력할 때 사용한다.

(2) DataInputStream, DataOutputStream

DataInputStream, DataOutputStream을 사용하면 기본 자료형 단위로 입출력할 수 있다.

(3) ObjectInputStream, ObjectOutputStream

ObjectInputStream, ObjectOutputStream는 객체 단위 입출력에 사용된다. 예제는 직렬화(Serialization) 포스트에서 확인할 수 있다.

스레드 동기화

멀티 스레드 환경에서는 여러 스레드가 한 데이터을 동시에 조작할 수 있다. 이 때문에 결과를 예측할 수 없게 될 수도 있다.

이 때문에 하나의 스레드가 데이터를 조작할 때 다른 스레드가 데이터를 조작할 수 없도록 공유데이터에 락(Lock)을 걸 수 있다. 이를 동기화(Synchronization)이라고 한다.

논리적으로는 출금(withdraw())을 호출할 때 출금액이 잔액보다 크면 잔액에서 출금을 마이너스하면 안된다.

그러나 두 스레드가 공유 데이터에 동시에 접근하기 때문에 잔액이 마이너스 값으로 출력된다. 따라서 동기화가 필요하다.

동기화는 두 가지 방법으로 할 수 있다.

(1) 방법 1

공유 데이터에 접근하는 메소드에 synchronized 키워드를 붙이면 된다.

특정 스레드가 synchronized메소드를 실행할 때 메소드 안에서 접근하는 데이터에 락(Lock)을 건다. 락이 걸린 상태에서 다른 스레드가 이 메소드를 호출하면 락이 풀릴 때까지 기다리게 된다.

(2) 방법 2

공유 데이터 자체에 synchronized 키워드를 붙일 수도 있다.

데드 락

동기화 과정에서 가장 조심해야할 부분이 데드락이다. 데드락(Deadlock)은 두 스레드가 서로 상대방이 락을 풀 때까지 기다리고 있기 때문에 결과적으로 아무것도 하지 못하는 상태를 가리킨다.

wait(), notify(), notifyAll()

Object 클래스에 정의된 wait(), notify(), notifyAll()을 사용하면 동기화의 효율을 높일 수 있다.

  • wait(): 한 스레드가 자원에 락을 걸고 오래 기다리는 대신 다른 스레드에게 제어권을 넘겨주고 대기 상태로 들어간다.
  • notify(): 다른 스레드는 작업을 하다가 notify()를 호출하며, 대기 상태에 들어갔던 스레드가 다시 실행 상태가 된다.

[Java] 스레드(Thread)의 상태

2018/04/01 카테고리 Java

Thread의 상태

스레드는 크게 네 가지 상태를 가지고 있다.

(1) Runnable

스레드를 생성하고 start()를 호출하면 Runnable상태가 된다.

주목할 점은 Runnable상태는 아직 스레드가 실행 중인 상태가 아니다.

(2) Running

Running은 스레드가 실행 중인 상태다.

Runnable과 Running상태를 구분한 이유는 여러 스레드가 존재할 수 있기 때문이다.

Thread클래스의 start()를 호출하면 Thread가 순서대로 Queue에 들어가며, 여기에서 Runnable상태로 대기하고 있다. 그리고 가장 먼저 들어온 Thread가 Running상태로 실행된다.

(3) Blocked

스레드가 일시중지 상태로 들어간다.

(4) Terminated

스레드가 종료되어 메모리에서 사라진다.
