Java Web Application은 일반적인 Java Application과는 여러가지 차이점이 있습니다. Eclipse나 InteliJ 같은 IDE를 사용하면 Java Web Application 프로젝트를 쉽게 생성할 수 있습니다. 그러나 이번 포스트에서는 직접 Java Web Application 프로젝트를 구성하며 프로젝트 구조에 대해 알아보겠습니다.

1. 프로젝트 구조

Java Web Application은 외부로 공개하는 디렉토리를 구성하고 그 안에 JSP 파일을 배치합니다. 또한 WEB-INF 디렉토리를 생성하고 그 안에 web.xml파일을 배치해야합니다.

maven과 maven에서 제공하는 maven-archtype-webapp 탬플릿을 사용하면 Java Web Application을 위한 디렉토리를 쉽게 구성할 수 있습니다.

2. Java 파일은 어디에 작성해야할까?

지금까지 생성한 프로젝트는 Java 프로젝트인데 java폴더가 없습니다. main폴더에 java폴더를 생성하고, 그 안에 패키지와 소스 코드 파일을 배치해나가면 됩니다.

[Java] 자바 플랫폼 종류

2018/01/02 카테고리 Programming, Java

자바 플랫폼은 크게 세 가지 에디션으로 나뉩니다.

1. Java SE (Java Platform, Standard Edition)

  • 가장 보편적으로 사용되는 에디션입니다.
  • 자바 데스크탑, 스마트폰 어플리케이션 개발에 사용됩니다.
  • 안드로이드 어플리케이션을 개발할 때에도 주로 Java SE가 사용됩니다.
  • 기초적인 데이터 타입부터 네트워킹, 보안, 데이터베이스 처리, GUI(Swing, AWT) 등 데스크탑 개발에 필요한 API를 포함합니다.

2. Java EE (Java Platform, Enterprise Edition)

  • 자바 서버 어플리케이션 개발에 사용됩니다.
  • Java SE의 모든 API를 포함합니다.
  • 그 외 EJB, JSP, Sevlet, JDBC 등 서버 어플리케이션 개발을 위한 API가 포함됩니다.
  • 2017년 오라클은 오픈소스 진영인 이클립스 재단에 Java EE를 넘겼으며, 명칭은 Jakarta EE로 변경되었습니다.

3. Java ME (Java Platform, Micro Edition)

  • Java SE API의 일부만 포함합니다.
  • 셋톱박스, PDA등 소형 임베디드 시스템을 개발하는데 사용됩니다.

[Java] 자바 컴파일러, JVM, JRE, JDK, Java Platform 차이

2018/01/01 카테고리 Programming, Java

자바 컴파일러

자바로 작성한 파일 이름 뒤에는 .java라는 확장자 붙습니다. 이 파일은 자바 소스코드라고 합니다. 자바 컴파일러는 이 소스코드를 자바 바이트코드로 변환합니다. 자바 바이트코드는 파일 이름 뒤에 .class확장자가 붙습니다.

JVM

JVM(Java Virtual Machine)은 자바 바이트코드를 운영체제에 종속적인 코드로 변환한 후 실행합니다.

서로 다른 운영체제라도 JVM만 설치되어 있다면, 같은 자바 어플리케이션을 다른 운영체제에서 실행할 수 있습니다.

단, JVM은 운영체제에 종속적이므로 각 운영체제에 맞는 JVM을 설치해야 합니다.

JRE

JRE(Java Runtime Edition)은 Java 애플리케이션의 실행에 요소들입니다. JRE는 다음 두 요소로 구성되어있습니다.

  • JVM
  • Java 애플리케이션 실행을 위한 API

JDK

JDK(Java Development Kit)는 Java 어플리케이션의 개발에 필요한 요소들입니다. JRE 외에도 Java Compiler, Java Debugger 등 개발에 필요한 도구들을 포함하고 있습니다.

Java SE 7, JDK 7, Java 7의 차이점

Java SE는 Java Platform Standard Edition의 약자로 Java Platform의 근간을 이루는 명세(Specification)를 가리킵니다.

JDK는 명세를 코드로 구현한 구현체(Implementation)입니다. 즉 Java SE 7은 JSR-336로 규정된 Java SE의 일곱 번째 명세를 의미하며 JDK 7은 OpenJDK JDK 7, Oracle JDK 7과 같은 구현체를 의미합니다.

추가적으로 Java 7은 엄밀히 말하자면 정확한 표현이 아닙니다. 하지만 Java SE 7이나 JDK 7보다 사용하기 편해서 흔히 Java 7이라고 부르는 것입니다.
