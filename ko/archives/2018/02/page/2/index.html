상속

실생활에서 상속은 자식이 부모의 재산을 물려받는 행위를 뜻한다. 자바에서도 상속은 비슷한 의미로 사용된다. 상속(Inheritance)은 자식 클래스가 부모 클래스의 멤버변수나 메소드를 그대로 물려받는 것을 의미한다. 이를 통해 코드의 중복을 제거할 수 있다.

사용 방법

다음과 같이 부모 클래스가 있다고 가정해보자.

부모 클래스를 상속할 땐 키워드extends를 사용한다. 자식 클래스에서는 @Override를 붙여서 부모 클래스의 메소드를 재정의할 수 있다.

주의할 점은 자식 클래스의 생성자에서 부모 클래스의 생성자를 호출해야 한다는 것이다. 이때는 키워드 super를 사용합니다.

문자열

Java에서 문자열은 원시 타입이 아니라 참조 타입이다. 문자열은 다음과 같이 생성한다

StringBuffer

이처럼 문자열은 Immutable Object이기 때문에 문자열 추가, 수정, 삭제 등의 연산이 빈번하면 메모리의 힙 영역에 객체가 여러 번 생성되어 성능에 영향을 주게 된다.

이러한 문제를 해결하기 위해 Java는 Mutable Object인 StringBuffer와 StringBuilder를 도입했다.

StringBuffer가 제공하는 append(), insert(), delete(), replace() 등의 메소드를 제공하면 동일한 객체에서 문자열을 변경할 수 있다.

StringBuffer클래스는 다양한 메소드를 지원한다.

(1) append()

끝에 문자열을 추가한다.

(2) insert()

특정 위치에 문자열을 추가한다.

(3) delete()

특정 범위의 문자열을 삭제한다.

(4) replace()

특정 범위의 문자열을 다른 문자열로 대치한다.

(5) reverse()

문자열을 뒤짚는다.

(6) substring()

문자열의 일부분을 추출한다.

(7) toString()

StringBuilder객체를 String객체로 변환한다.

StringBuilder

StringBuilder는 StringBuffer와 사용법이 거의 유사하다. 다만 다음과 같은 차이점이 있다.

StringBuffer StringBuilder
동기화를 지원한다 동기화를 지원하지 않는다
멀티 스레드 환경에서 안전하다 멀티 스레드 환경에서 안전하지 않다.
동기화 때문에 성능이 상대적으로 낮다. 성능이 상대적으로 높다.

따라서 단일 스레드 환경에서는 StringBuilder를 사용하고 멀티 스레드 환경에서는 StringBuffer를 사용한다.

요약하자면 문자열의 추가, 수정, 삭제가 빈번한 경우 String보다 StringBuffer 또는 StringBuilder클래스를 사용한다.

String 클래스

String클래스에는 문자열 조작을 위한 다양한 메소드가 존재한다.

(1) toUpperCase()

문자열을 대문자로 변환한다.

(2) toLowerCase()

문자열을 소문자로 변환한다.

(3) substring()

문자열의 일부분을 반환한다.

(4) trim()

문자열 앞과 뒤의 공백을 제거한다.

(5) spilt()

인자로 전달받은 정규표현식 또는 문자를 기준으로 문자열을 나누어 배열에 저장하여 반환한다.

(6) replace()

특정 문자를 다른 문자로 대치한다.

(7) replaceAll()

특정 문자열을 다른 문자열로 대치한다.

(8) startsWith()

문자열이 특정 문자열로 시작하는지 확인한다.

(9) endsWith()

문자열이 특정 문자열로 끝나는지 확인한다.

(10) length()

문자열의 길이를 반환한다.

(11) indexOf()

문자열에서 특정 문자의 인덱스를 반환한다. 특정 문자가 없으면 음수값을 반환한다.

(12) charAt()

문자열에서 특정 인덱스에 위치하는 문자를 반환한다.

(13) concat()

문자열 뒤에 추가한다. +연산자와 동일하다.

(14) equals()

두 문자열이 동일한지 비교한다.

(15) contains()

특정 문자나 문자열이 포함되는지 확인한다.

(16) matches()

문자열이 특정 패턴의 문자열을 포함하는지 확인한다. 특정 패턴은 정규 표현식을 의미한다.

static

키워드 static은 클래스의 멤버변수와 메소드 앞에 붙일 수 있다. 이 키워드가 붙어있는 멤버변수와 메소드를 정적 멤버변수와 정적 메소드라고 한다.

정적 멤버변수와 정적 메소드는 다음과 같은 특징이 있다.

  • 클래스의 인스턴스를 생성하지 않고도 정적 메소드를 호출할 수 있다.
  • 클래스의 인스턴스를 생성하지 않고도 정적 멤버변수에 접근할 수 있다.
  • 클래스의 모든 인스턴스가 정적 멤버변수를 공유한다.

Collections 클래스

Collections클래스는 Collection인터페이스를 구현한 클래스를 조작할 수 있는 다양한 메소드를 제공한다. 이 클래스를 사용하여 정렬, 병합, 검색 등의 다양한 작업을 수행할 수 있다.

  • max()
  • min()
  • sort()
  • reverse()
  • shuffle()
  • binarySearch()
  • disjoint()

max()

max()는 Collection에서 최대값을 찾아 반환한다.

min()

max()는 Collection에서 최소값을 찾아 반환한다.

sort()

sort()는 Collection을 정렬하는데 사용한다.

reverse()

reverse()는 배열을 뒤집는다.

shuffle()

shuffle()은 배열의 요소를 무작위로 섞는다.

binarySearch()

binarySearch()는 정렬된 배열에서 요소를 탐색한다. 요소가 배열에 있는 경우 인덱스를 반환한다. 요소가 배열에 없는 경우 음수값을 반환한다.

disjoint()

disjoint()는 두 Collection에서 일치하는 요소가 하나도 없으면 true, 하나라도 있으면 false를 반환한다.

(1) 사용법

PriorityQueue는 데이터를 Enqueue한 순서에 상관없이 우선순위가 높은 데이터가 먼저 Dequeue되는 자료구조다.
