컬렉션 프레임워크

Java API는 같은 타입의 여러 데이터를 한꺼번에 효율적으로 관리하기 위해 컬렉션 프레임워크(Collection Framework)를 지원한다.

컬렉션 프레임워크는 java.util 패키지에 포함되어있으며, 가장 중요한 요소는 List, Set, Map이다.

List

배열(Array)은 생성할 때 크기가 결정된다. 따라서 배열이 생성되면 동적으로 데이터를 추가하거나 삭제할 수 없다. 데이터를 동적으로 추가, 삭제하려면 List를 사용해야한다.

Vector vs. ArrayList

Vector는 동기화가 되어있다. 다시 말해 한 순간에 하나의 스레드만 접근할 수 있기 때문에 스레드 안전(Thread Safe)이라고 한다. 반면 ArrayList는 동기화되지 않았기 때문에 동시에 여러 스레드가 접근할 수 있다.

Vector는 동기화가 되어있는 대신 속도가 느리다. 반면 ArrayList는 동기화가 되지 않았기 때문에 속도가 빠르다.

따라서 싱글 스레드 환경에서는 ArrayList를 사용하고 멀티 스레드 환경에서는 Vector를 사용하는 것이 바람직하다.

ArrayList vs. LinkedList

ArrayList는 내부적으로 배열을 사용한다.

ArrayList를 생성할 때 크기가 고정된 배열을 생성한다. 그리고 고정된 배열이 꽉 차면 더 큰 크기의 새로운 배열을 생성하고 복사한다.

반면 LinkedList는 내부적으로 Node를 사용한다.

보통 데이터를 순차적으로 추가, 삭제하는 경우 ArrayList가 더 빠르다. 반면 데이터를 중간에 추가, 삭제하는 경우 LinkedList가 더 빠르다. 따라서 상황에 따라 적절한 것을 선택하자.

[Java] Arrays 클래스

2018/02/07 카테고리 Java

Arrays 클래스

Arrays클래스에는 배열을 조작하기 위한 다양한 메소드가 정의되어있다.

  • toString()
  • sort()
  • fill()
  • setAll()
  • copyOf()
  • copyOfRange()
  • equals()
  • deepEquals()
  • asList()

toString()

toString()은 배열을 문자열로 반환한다.

sort()

sort()는 배열을 정렬한다.

요소가 객체인 경우 Comparable과 Comparator를 사용한다.

binarySearch()

binarySearch()는 배열에서 요소를 이진 탐색한다. 다만 배열이 정렬되어있어야 한다.

fill()

fill()은 배열의 모든 요소를 동일한 값으로 채운다.

setAll()

setAll()은 함수형 인터페이스로 배열을 채운다.

copyOf()

copyOf()는 배열을 깊은 복사할 때 사용한다.

copyOfRange()

copyOfRange()는 배열의 일부 요소만을 깊은 복사할 때 사용한다.

equals()

equals()은 1차원 배열을 비교할 때 사용한다.

deepEquals()

deepEquals()은 다차원 배열을 비교할 때 사용한다.

asList()

asList()를 사용하면 배열을 List로 변환할 수 있다.

참고로 Arrays.asList()로 생성한 List은 새로운 원소를 추가하거나 삭제할 수 없다.

따라서 원소를 추가하거나 삭제하려면 ArrayList클래스를 사용해야한다.

[Java] Wrapper 클래스

2018/02/05 카테고리 Java

Wrapper 클래스

개발을 하다보면 원시 타입(Primitive Type)을 객체로 취급해야하는 경우가 있다. 예를 들면 Java에서 Generic을 사용할 때는 타입 파라미터로 참조 타입만을 전달할 수 있다.

이 때는 Wrapper 클래스를 사용하면 된다. 즉 Wrapper 클래스는 원시 타입을 참조 타입으로 취급하기 위해 사용한다.

Wrapper 클래스는 java.lang 패키지에 포함되어있으며, 원시 타입에 대응하는 Wrapper 클래스가 존재한다.

원시 타입 래퍼 클래스
byte Byte
char Character
short Short
int Integer
long Long
boolean Boolean
float Float
double Double

박싱과 언박싱

원시 타입의 값으로 Wrapper 클래스를 만드는 것을 박싱(Boxing)이라고 한다.

Wrapper 클래스에서 기본 타입의 값을 얻어내는 것을 언박싱(Boxing)이라고 한다.

오토 박싱과 오토 언박싱

직접적으로 박싱/언박싱을 하지 않아도 자동으로 박싱/언박싱이 일어나기도 한다. Wrapper 클래스에 원시 타입의 데이터를 할당할 경우 오토 박싱이 된다.

Wrapper 클래스의 객체를 원시 타입에 할당할 때도 오토 언박싱이 된다.

[Java] Java 유료화 논쟁 - Oracle JDK, OpenJDK

2018/01/02 카테고리 Java

들어가며

2018년 Oracle이 Java SE의 과금 방식을 영구 라이선스에서 구독 모델로 변경하겠다고 선언했다. 이 선언이 그동안 무료로 인식된 Java의 유료화로 받아들여지면서 관련 업계에 큰 혼란을 초래하고있다. Java의 유료화를 이해하려면 Oracle JDK와 OpenJDK에 대해 알아야 한다.

참고로 Java 언어 자체는 무료다. Java 언어로 작성된 어플리케이션을 컴파일하고 실행하는 JDK(Java Development Kit)가 유료화 논쟁의 대상이다.

Oracle JDK

Oracle JDK는 사용 목적에 따라 유료이기도 무료이기도 한 JDK다.

Oracle JDK는 서버, PC, 스마트폰 등의 일반적인 목적으로 사용될 때는 무료다. 반면 키오스크 등 특수한 목적의 컴퓨팅에 사용될 땐 유료다. 또한 Oracle JDK 8(Oracle Java SE 8)부터는 Oracle의 지속적인 업데이트와 보안 패치, 기술지원을 받으려면 유료 서비스를 구매해야한다. (참고로 Oracle JDK 8 이전 버전에서도 일반적인 목적의 사용은 무료였고 지속적인 업데이트와 보안 패치, 기술지원은 유료였다.)

Oracle JDK는 OpenJDK를 기본으로 한다. Open JDK에 추가적인 기능이 더해진 것이 Oracle JDK다. Oracle JDK에 존재하고 Open JDK에는 없는 대표적 기능으로는 글꼴 라이브러리와 Java Web Start다.

OpenJDK

OpenJDK는 사용 목적에 관계없이 무료인 JDK다.

2007년 Java를 개발한 Sun Microsystems사는 JDK를 오픈소스화하기 위해 OpenJDK를 만들었다. Sun Microsystems사는 저작권자가 오픈소스화를 거부한 일부 기능을 제외한 나머지 JDK 소스코드 전부를 OpenJDK에 제공했고, OpenJDK는 이를 기반으로 JDK 7 프로젝트를 시작했다.

JDK 6 시절에는 OpenJDK가 Oracle JDK보다 성능이나 안정성이 크게 떨어졌다. 하지만 오늘날에는 Oracle JDK만이 제공하는 일부 기능을 제외하고는 차이가 없다는게 Oracle의 설명이다.

안드로이드 스튜디오의 JDK

참고로 안드로이드 스튜디오에는 OpenJDK가 내장되어 있으므로 별도의 JDK 설치가 필요없다. 설치 경로는 File > Project Structure > SDK Location에서 확인할 수 있다.

안드로이드 스튜디오에서는 보통 compileSDKVersion에 따라 적당한 JDK 버전이 자동으로 설정된다. 물론 프로젝트 수준의 build.gradle 파일에서 직접 JDK 버전을 설정할 수 있다.

결론

Kakao나 Line처럼 한국에서 잘 알려진 기업도 OpenJDK를 도입하고 활용하고 있는 것으로 알려져있다. OpenJDK에서는 제공하지 않는 컴포넌트를 사용해야하거나, Oracle의 지속적인 지원을 받아야 하거나, 키오스크 등 특수한 목적의 컴퓨팅에 사용하는 경우에는 비용을 지불하고 Oracle JDK를 사용해야 할 것이다. 그렇지 않은 경우에는 OpenJDK를 사용해도 무방할 것 같다.
