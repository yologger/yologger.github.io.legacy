스레드 동기화

멀티 스레드 환경에서는 여러 스레드가 한 데이터을 동시에 조작할 수 있다. 이 때문에 결과를 예측할 수 없게 될 수도 있다.

이 때문에 하나의 스레드가 데이터를 조작할 때 다른 스레드가 데이터를 조작할 수 없도록 공유데이터에 락(Lock)을 걸 수 있다. 이를 동기화(Synchronization)이라고 한다.

논리적으로는 출금(withdraw())을 호출할 때 출금액이 잔액보다 크면 잔액에서 출금을 마이너스하면 안된다.

그러나 두 스레드가 공유 데이터에 동시에 접근하기 때문에 잔액이 마이너스 값으로 출력된다. 따라서 동기화가 필요하다.

동기화는 두 가지 방법으로 할 수 있다.

(1) 방법 1

공유 데이터에 접근하는 메소드에 synchronized 키워드를 붙이면 된다.

특정 스레드가 synchronized메소드를 실행할 때 메소드 안에서 접근하는 데이터에 락(Lock)을 건다. 락이 걸린 상태에서 다른 스레드가 이 메소드를 호출하면 락이 풀릴 때까지 기다리게 된다.

(2) 방법 2

공유 데이터 자체에 synchronized 키워드를 붙일 수도 있다.

데드 락

동기화 과정에서 가장 조심해야할 부분이 데드락이다. 데드락(Deadlock)은 두 스레드가 서로 상대방이 락을 풀 때까지 기다리고 있기 때문에 결과적으로 아무것도 하지 못하는 상태를 가리킨다.

wait(), notify(), notifyAll()

Object 클래스에 정의된 wait(), notify(), notifyAll()을 사용하면 동기화의 효율을 높일 수 있다.

  • wait(): 한 스레드가 자원에 락을 걸고 오래 기다리는 대신 다른 스레드에게 제어권을 넘겨주고 대기 상태로 들어간다.
  • notify(): 다른 스레드는 작업을 하다가 notify()를 호출하며, 대기 상태에 들어갔던 스레드가 다시 실행 상태가 된다.

[Java] 스레드(Thread)의 상태

2018/04/01 카테고리 Java

Thread의 상태

스레드는 크게 네 가지 상태를 가지고 있다.

(1) Runnable

스레드를 생성하고 start()를 호출하면 Runnable상태가 된다.

주목할 점은 Runnable상태는 아직 스레드가 실행 중인 상태가 아니다.

(2) Running

Running은 스레드가 실행 중인 상태다.

Runnable과 Running상태를 구분한 이유는 여러 스레드가 존재할 수 있기 때문이다.

Thread클래스의 start()를 호출하면 Thread가 순서대로 Queue에 들어가며, 여기에서 Runnable상태로 대기하고 있다. 그리고 가장 먼저 들어온 Thread가 Running상태로 실행된다.

(3) Blocked

스레드가 일시중지 상태로 들어간다.

(4) Terminated

스레드가 종료되어 메모리에서 사라진다.

[Gradle] Compile과 Build

2018/04/01 카테고리 Tools, Build Tool, Gradle

Compile

컴파일(Compile) 개발자가 작성한 소스코드를 컴퓨터가 이해할 수 있는 기계어로 변환하는 과정을 의미한다.

예를 들어 C 언어의 경우 gcc컴파일러가 .c파일을 컴파일하여 .o파일을 생성한다.
Java 언어의 경우 javac컴파일러가 .java파일을 컴파일하여 .class파일을 생성한다.

Build

프로젝트를 진행하다보면 여러 개의 소스코드 파일이 생성된다. A라는 소스코드 파일에서 B라는 소스코드 파일을 참조한다고 가정해보자. 이때 파일 A와 파일 B를 각각 컴파일하면 A에서 B를 찾지 못하므로 참조할 수가 없다. 따라서 파일 A와 파일 B를 연결하는 작업이 필요한데 이를 링크(Link)라고 합니다.

빌드(Build)는 컴파일, 링크 등을 모두 포함하여 실행 가능한 소프트웨어로 만드는 과정을 의미합니다.

Build Tool

빌드 도구(Build Tool)는 소스코드를 실행 가능한 소프트웨어로 만들어주는 도구입니다. 요즘의 빌드 도구는 빌드 작업 뿐만 아니라 테스트, 빌드 자동화, 배포 등 다양한 기능을 제공하기도 합니다. 자바나 코틀린의 빌드 도구에는 Maven, Build 등이 있습니다.
