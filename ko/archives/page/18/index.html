[Kotlin] 데이터 클래스(Data Class)

2018/08/17 카테고리 Kotlin

데이터 클래스

Kotlin은 데이터만을 담기 위한 클래스를 제공한다. 이를 데이터 클래스(data class)라고 한다. 데이터 클래스를 선언할 때는 키워드data를 붙인다. 주의할 점은 멤버 변수를 반드시 기본 생성자에 추가해야한다.

데이터 클래스는 데이터를 처리하는데 유용한 메소드들을 자동으로 제공한다. 자주 사용하는 몇 가지 메소드만 알아보자.

(1) equal()

equal()메소드는 두 인스턴스가 동일한지 비교하는데 사용한다.

보통 equal()은 개발자가 정의하여 사용한다.

(2) toString()

toString()메소드는 인스턴스를 문자열로 변환하는데 사용한다.

보통 toString()도 개발자가 정의하여 사용한다.

(3) copy()

copy()메소드는 인스턴스를 복사하는데 사용한다.

[Kotlin] 인터페이스(Interface)

2018/08/16 카테고리 Kotlin

인터페이스

모든 메소드가 선언만 있고 구현부가 없는 클래스를 인터페이스(Interface)라고 한다. 인터페이스를 선언할 때는 키워드 interface를 사용한다.

인터페이스는 인스턴스를 만들 수 없으며, 이를 구현한 클래스를 작성해야한다. 인터페이스를 구현한 클래스를 구현체(Implementation)라고 한다.

인터페이스를 구현한 클래스는 인스턴스를 생성할 수 있다.

[Kotlin] 추상 클래스(Abstract Class)

2018/08/15 카테고리 Kotlin

추상 클래스

선언만 있고 구현부는 없는 메소드를 추상 메소드(Abstract Method)라고 한다. 또한 추상 메소드를 포함하는 클래스를 추상 클래스(Abstract Class)라고 한다. 추상 메소드와 추상 클래스는 앞에 키워드 abstract를 붙인다.

추상 클래스는 인스턴스를 생성할 수 없다.

추상 클래스를 상속하는 자식 클래스에서는 추상 메소드를 재정의할 수 있다. 이를 오버라이드(override)라고 한다.
