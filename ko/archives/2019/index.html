Spring IoC Container의 생명주기

지금까지는 다음과 같은 방법으로 Spring IoC Container를 생성했습니다.

이 것을 풀어서 작성하면 다음과 같습니다.

Bean의 생명주기

Spring IoC Container는 Bean의 생성, 사용, 소멸을 제어하고 관리하며, Bean은 생명주기를 가지고 있다고 합니다. Bean의 생명주기는 두 가지 방법으로 탐지할 수 있습니다.

방법 1) InitializingBean, DispoableBean

InitializingBean, DispoableBean 인터페이스를 구현하고 afterPropertiesSet(), destroy() 메소드를 구현하면 Bean의 생성과 소멸을 탐지할 수 있습니다.

방법 2) @PostConstruct, @PreDestory

@PostConstruct어노테이션을 추가한 메소드는 Bean이 생성될 때 호출됩니다. 비슷하게 @PreDestory어노테이션을 추가한 메소드는 Bean이 소멸할 때 호출합니다.

Scope

Bean은 Scope를 가집니다. 기본값은 prototype이며 빈을 주입할 때 마다 새로운 인스턴스가 생성됩니다.

scope를 singleton로 설정하면 Bean은 IoC Container내에서 오직 한 개만 존재합니다.

[Servlet, JSP] Servlet Filter란?

2019/01/14 카테고리 Spring, Servlet, JSP

Filter

Filter를 사용하면 Client에서 온 요청을 Servlet에서 처리하기 전에 필터링할 수 있습니다. Filter는 다음과 같은 용도로 활용할 수 있습니다.

  • 사용자 인증 필터
  • 로깅 또는 감시 필터
  • 이미지 변환 및 압축
  • 데이터 변화

Filter는 Filter인터페이스를 구현하고 세 가지 메소드를 구현해야합니다.

(1) init()

Filter가 처음 생성될 때만 한 번 호출됩니다.

(2) destroy()

Filter가 소멸될 때 한 번 호출됩니다.

(3) doFilter()

이 메소드에서 Client의 Request를 필터링합니다.

이 함수의 세 번째로 FilterChain 객체가 전달됩니다. 여러 개의 필터가 설정된 경우 FilterChain클래스의 doFilter()를 호출해야 다음 필터로 넘어갑니다.

Filter 등록

Filter는 두 가지 방법으로 등록할 수 있습니다.

(1) @WebFilter 어노테이션

@WebFilter을 사용하면 됩니다. 이때 필터링할 경로를 함께 전달합니다. 모든 경로에 대한 Request에 필터링을 하려면 /*을 전달하면 됩니다.

(2) web.xml에 등록

web.xml에 필터를 등록할 수도 있습니다.

[Servlet, JSP] JSP란?

2019/01/11 카테고리 Spring, Servlet, JSP

JSP

JSP(Java Server Pages)는 HTML에 Java 코드를 삽입하여 동적으로 웹 페이지를 생성하는 언어입니다. 아래는 JSP의 예제입니다.

JSP의 동작 원리

JSP는 다음과 같은 과정으로 처리됩니다.

  1. Web container는 JSP 파일을 Servlet으로 변환합니다.
    • MyPage.jsp > MyPage_jsp.java
  2. Servlet 내부에서 JSP 파일의 내용을 출력합니다. MyPage_jsp.java의 코드 일부를 살펴봅시다.
  3. Web Container는 Servlet을 처리합니다.

요약하자면 JSP는 MyPage.js > MyPage_jsp.java > MyPage_jsp.class > HTML 순서대로 처리됩니다.

JSP tag

JSP tag는 JSP 문서 안에 Java 코드를 삽입할 때 사용합니다.

1. 자바 코드

<% %>를 사용하여 자바 코드를 추가할 수 있습니다.

2. 주석

주석은 <%-- --> 형태로 사용합니다.

3. 메소드, 변수 선언

<%! %>는 변수를 선언하거나 함수를 정의할 때 사용합니다.

4. 표현식

JSP에서 무언가를 출력하려면 out.println()을 사용해야합니다.
<%! %>는 변수를 선언하거나 함수를 정의할 때 사용합니다.

<%= %>를 사용하면 더 쉽게 출력할 수 있습니다.

5. 지시자

지시자는 JSP 페이지의 속성을 지정할 때 사용합니다. <%@ %> 형태로 사용합니다.

(1-1) page

page 속성은 해당 페이지의 속성을 지정할 때 사용합니다.

(1-2) include

페이지에 다른 페이지를 삽입할 때 사용합니다.

(1-3) taglib

태그 라이브러리를 사용할 때 사용합니다.

JSP Action Tag

JSP Action Tag는 JSP 페이지 내에서 특정 동작을 하도록 지시하는 태그입니다.

(1) include 태그

현재 페이지에 다른 페이지를 삽입합니다.

(2) forward 태그

현재 페이지에서 다른 페이지로 전환합니다.

(3) param 태그

forward 태그나 include 태그를 사용할 때 데이터를 전달하기 위해 사용합니다.

[Servlet, JSP] ServletContextListener

2019/01/10 카테고리 Spring, Servlet, JSP

ServletContextListener

ServletContextListener을 사용하면 웹 애플리케이션의 생명주기를 감시할 수 있습니다. 우선 ServletContextListener클래스를 상속하는 클래스를 정의합니다.

ServletContextListener의 contextInitialized(), contextDestroyed() 메소드를 오버라이드합니다.

  • contextInitialized(): Web Application이 시작할 때 해당 메소드가 호출됩니다.
  • contextDestroyed(): Web Application이 소멸할 때 해당 메소드가 호출됩니다.

이제 AppListener를 Web Container에 등록해야합니다. 두 가지 방법이 있습니다.

방법 1. web.xml

<listener>태그를 사용하여 Web.xml에 등록합니다.

방법 2. @WebListener 어노테이션

ServletContextListener를 상속하는 클래스에 @WebListener어노테이션을 추가합니다.
