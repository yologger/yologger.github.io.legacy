[Servlet, JSP] Servlet 이란?

2019/01/08 카테고리 Spring, Servlet, JSP

Servlet

Servlet은 Java를 사용하여 웹 페이지를 동적으로 생성하는 서버측 컴포넌트입니다. 모든 Servlet은 HttpServlet클래스를 상속합니다.

Servlet은 두 가지 방법으로 URL Mapping을 할 수 있습니다.

방법 2. web.xml

Servlet의 동작 순서


  1. Client로 부터 요청이 들어오면 WAS는 Web Container와 Context를 생성합니다.
  2. Web Container는 요청에 대한 HttpServletRequest, HttpServletResponse를 생성합니다.
  3. Web Container는 web.xml의 Servlet 매핑 정보를 확인하여 어떤 Servlet을 생성할지 결정합니다.
  4. Web Container는 Servlet이 한 번도 실행된 적이 없으면 Servlet의 인스턴스를 생성합니다. 실행된 적이 있으면 Servlet을 재사용합니다.
  5. Web Container는 요청을 처리하기 위한 Thread를 생성합니다.
  6. Servlet의 doGet(), doPost(), service()가 호출되며 인자로 HttpServletRequest, HttpServletResponse 객체가 전달됩니다.
  7. 로직이 수행되면 HttpServletRequest, HttpServletResponse 객체를 소멸시키고 Thread를 종료합니다.

Servlet의 생명주기

Web Container는 Client의 요청에 따라 Servlet을 생성, 관리, 소멸합니다. 이때 Servlet 클래스에 정의된 생명주기 함수가 호출됩니다.


  • Constructor: Servlet 인스턴스가 생성될 때 최초 한 번 호출되는 생성자입니다.
  • init(): Servlet 인스턴스가 생성될 때 최초 한 번 호출되는 생명주기 함수입니다.
  • doGet(), doPost(), service(): Client에서 요청할 때 마다 호출되는 생명주기 함수입니다.
    • doGet(): HTTP GET 방식으로 요청할 때 호출됩니다.
    • doPost(): HTTP POST 방식으로 요청할 때 호출됩니다.
    • service(): doGet()과 doPost()가 정의되지 않았을 때 호출됩니다.
  • destroy(): Servlet 인스턴스가 소멸될 때 한번 호출 됩니다.

Servlet 선처리, 후처리

@PostConstruct, @PreDestroy 어노테이션을 임의의 메서드에 추가하여 선처리, 후처리를 할 수 있습니다.

doGet()

HTTP GET 방식으로 요청을 하면 doGet()메소드가 호출됩니다.

doPost()

HTTP POST 방식으로 요청하면 doPost()메소드가 호출됩니다.

[Servlet, JSP] Server, Web Server, WAS 차이점

2019/01/04 카테고리 Spring, Servlet, JSP

Server

Server는 Client에 정보나 서비스를 제공하는 컴퓨터 시스템입니다. 서버에는 보통 Linux, Unix, Windows 운영체제가 탑재됩니다.

Web Server

  • Web Server는 웹 브라우저 같은 클라이언트로부터 HTTP 요청을 받아들이고 HTML 문서와 같은 웹 페이지를 반환하는 프로그램입니다.
  • Web Server의 대표적인 제품군에는 Apache, Nginx, IIS가 있습니다.
  • Web Server는 파일 시스템에 저장된 파일(HTML, CSS, JavaScript, 이미지, 파일 등)을 단순히 클라이언트에게 전달합니다. 이를 정적 컨텐츠를 제공한다고 합니다.

WAS(Web Application Server)

WAS는 데이터베이스, 파일 시스템, 네트워크 등에서 데이터를 읽어와 다양한 로직을 수행한 후 적절한 HTML 파일을 생성하여 클라이언트에게 전달합니다. 이를 동적 컨텐츠를 제공한다고 합니다.

WAS의 대표적인 제품군에는 Apache Tomcat, Oracle WebLogic, IBM Websphere, Tmax JEUS, Red Hat Jboss가 있습니다. 제품군에 따라 다르지만 WAS는 크게 세 가지 요소로 구성됩니다.

  • Web Container
    1. JSP파일을 Servlet으로 변환합니다.
    2. Servlet의 생성, 소멸 등의 생명주기를 관리합니다.
    3. HttpServletRequest, HttpServletResponse의 생성, 소멸 등의 생명주기를 관리합니다.
    4. Web Container는 Servlet Container라고도 불립니다.
  • Web Server: Web Container가 동적으로 생성한 웹 페이지를 Client에게 제공합니다.
  • EJB Container:
    1. EJB(Enterprise Java Bean)의 생명주기를 관리합니다.
    2. EJB는 한국에서는 Spring Framework에 밀려서 잘 사용되지 않습니다.

Oracle WebLogic, Red Hat Jboss 같은 제품군은 세 가지 요소를 모두 포함합니다. 그러나 Apache Tomcat은 Web Server와 Web Container만을 포함합니다.

2019/01/02 카테고리 Spring, Servlet, JSP

Eclipse를 실행하고 File > Project를 선택한 후 Dynamic Web Project를 선택합니다.

Project name, Project Location, Target runtime을 적절히 입력하고 Next > 버튼을 누릅니다.

Source folders on build path는 Java 파일이 위치할 경로입니다. Default output folder는 빌드된 파일이 위치할 경로입니다. 확인 후 Next >버튼을 누릅니다.

Context root와 Content directory은 기본값으로 두고 Generate web.xml deployment descriptor를 선택하고 Next >버튼을 누릅니다.

  • Context root: WAS(Tomcat)에는 여러 개의 Web Application이 구동될 수 있습니다. Context root는 WAS위의 Web Application을 구분하는데 사용됩니다. 예를 들어 Web Appication의 URL이 http://yourcompany.com/my_project라면 my_project가 context root를 의미합니다.
  • Content directory: JSP파일이 저장되는 경로입니다. JSP파일 외에도 META-INF폴더, WEB-INF폴더, web.xml파일 등이 위치합니다.
  • enerate web.xml deployment descriptor: WEB.xml 파일을 생성합니다.

[안드로이드] APK 파일이란?

2019/01/01 카테고리 Android, Android Platform

APK

APK(Android Application Package)는 안드로이드 어플리캐이션 설치 파일입니다. 안드로이드 시스템에서 APK 파일을 사용하여 어플리케이션을 설치할 수 있습니다. Windows 계열의 운영체제에서의 인스톨러와 유사합니다.

APK 생성 과정

APK 파일은 다음과 같은 과정을 거쳐 생성됩니다.

  1. 안드로이드 프로젝트는 소스코드 및 리소스로 구성된 모듈과 의존성으로 구성됩니다.
  2. Gradle은 이 모든 것을 빌드하여 DEX 파일과 컴파일된 리소스를 생성합니다.
  3. APK Manager는 개인키로 서명하여 APK 파일를 생성합니다.
  4. APK 파일은 구글 플레이스토어에 업로드하거나 안드로이드 시스템에 직접 설치할 수 있습니다.

[Servlet, JSP] Maven으로 Java Web Application 구성하기

2019/01/01 카테고리 Spring, Servlet, JSP

Java Web Application은 일반적인 Java Application과는 여러가지 차이점이 있습니다. Eclipse나 InteliJ 같은 IDE를 사용하면 Java Web Application 프로젝트를 쉽게 생성할 수 있습니다. 그러나 이번 포스트에서는 Maven으로 Java Web Application을 구성하면서 Java Application과의 차이점, 그리고 프로젝트의 구조에 대해 알아보겠습니다.

1. 프로젝트 구조

Java Web Application은 외부로 공개하는 디렉토리를 구성하고 그 안에 JSP 파일을 배치합니다. 또한 WEB-INF 디렉토리를 생성하고 그 안에 web.xml파일을 배치해야합니다.

maven과 maven에서 제공하는 maven-archtype-webapp 탬플릿을 사용하면 Java Web Application을 위한 디렉토리를 쉽게 구성할 수 있습니다.

생성된 프로젝트의 구조는 다음과 같습니다.

webapp폴더에는 JSP 파일과 WEB-INF 폴더가 위치합니다. 이 webapp폴더가 외부로 공개되며, 웹 브라우저에서 다음과 같이 접근할 수 있습니다.

2. Java 파일은 어디에 작성해야할까?

지금까지 생성한 프로젝트는 Java 프로젝트인데 java폴더가 없습니다. 이때는 main폴더에 java폴더를 생성한 후, 그 안에 패키지와 소스 코드 파일을 배치해나가면 됩니다.

3. 의존성 추가

JSP, Servlet 등 Java Web Application의 핵심 요소들을 사용하려면 의존성을 추가해야합니다.

4. Web Application Server 준비

일반적인 Java Application은 JDK만 준비되면 쉽게 실행할 수 있습니다. 그러나 Java Web Application은 JDK 외에도 WAS(Web Application Server)가 필요합니다. Apache Tomcat이 대표적인 WAS 입니다.
