[Kotlin] 제너릭(Generic)

2018/09/15 카테고리 Kotlin

제네릭

제네릭(Generic)은 클래스나 메서드 내부에서 사용할 데이터의 자료형을 외부에서 설정하는 것입니다. 이를 통해 컴파일 단계에서 메서드나 클래스에 사용되는 다양한 데이터의 타입 체크를 가능하게 합니다.

제네릭 클래스

제네릭 클래스를 정의할 땐 <>안에 임의의 문자를 명시합니다. 이 문자를 타입 파라미터(Type Parameter)라고 합니다. 타입 파라미터는 관습적으로 대문자 E(Element), T(Type), K(Key), V(Value)를 사용합니다.

제네릭 함수

제네릭은 클래스 뿐만 아니라 함수에도 사용할 수 있습니다.

(1) 함수의 파라미터

우선 함수에서 제네릭을 사용할 땐 키워드 fun과 함수명 사이에 타입 파라미터를 명시합니다.

(2) 반환 값

함수의 반환 값에도 타입 파라미터를 사용할 수 있습니다.

키워드 where를 사용해도 타입 파라미터에 실제로 오는 자료형을 제한할 수 있습니다. 아래 코드를 살펴봅시다.

in

키워드in은 자바 제네릭의 super와 유사합니다. 키워드in은 제네릭의 타입 파라미터를 특정 클래스와 그 클래스의 조상 클래스로 제한합니다.

out

키워드out은 Java 제너릭의 extends와 유사합니다. 키워드out는 타입 파라미터의 실제 타입을 특정 클래스와 특정 클래스의 자손 클래스로 제한합니다.
