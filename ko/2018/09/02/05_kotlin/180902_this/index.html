[Kotlin] Kotlin에서의 this

2018/09/02 카테고리 Kotlin

(1) 클래스에서의 this

클래스에서의 this는 해당 위치에서 가장 가까운 클래스의 인스턴스를 의미합니다. 아래 예제를 살펴봅시다.

위 코드에서의 this는 Person클래스의 인스턴스를 의미합니다. 키워드 this는 생략할 수도 있습니다.

(2) 확장 함수에서의 this

확장 함수에서의 this는 확장 함수를 호출한 클래스의 인스턴스를 의미합니다. 다음과 같은 클래스가 있다고 가정합시다.

확장 함수를 추가해봅시다.

확장 함수에서 this는 점(.)의 왼쪽에 위치하는 확장 함수를 호출한 클래스 (예제에서는 Person클래스의 인스턴스)를 의미합니다.

(3) 스코프 함수에서의 this

스코프 함수에서의 this는 리시버를 의미합니다. 리시버(Receiver)는 스코프 호출을 호출한, 점(.)의 왼쪽에 위치하는 대상입니다.

우선 스코프 함수 run() 예제를 살펴봅시다.

스코프 함수 with()의 예제도 살펴봅시다.

(4) Qualified this

아래 코드를 살펴봅시다. 클래스 안에 내부 클래스가 있고, 내부 클래스 안에 확장 함수가 있습니다.

이때 확장 함수에서의 this는 무엇을 가리킬까요?

확장 함수에서의 this는 확장 함수를 호출하는 리시버인 Int입니다.

그럼 확장 함수 안에서 Outer클래스와 Inner클래스는 어떻게 접근할 수 있을까요? 이러한 경우에 Qualifed this를 사용합니다. Qualifed this는 this@클래스이름 형태로 사용합니다.

아래 코드는 안드로이드에서 자주 사용하는 패턴으로 버튼을 눌렀을 때 특정 로직을 수행하는 코드입니다.

onClick()메소드 안에서의 this는 View.OnClickListener를 가리킵니다. 따라서 액티비티의 인스턴스를 참조하기 위해 this@MyActivity를 사용합니다.
